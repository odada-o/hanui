# Chrome DevTools MCP 명령어

Chrome DevTools MCP를 사용하여 브라우저 자동화, 디버깅, 성능 분석을 수행합니다.

## 사용 전 준비사항

Chrome을 원격 디버깅 모드로 실행해야 합니다:

```bash
./scripts/start-chrome-debug.sh
```

## 주요 기능

### 1. 성능 분석

- **LCP (Largest Contentful Paint) 측정**
  - 페이지의 주요 콘텐츠 렌더링 시간 측정
  - Core Web Vitals 지표 확인

- **페이지 로드 성능 분석**
  - 전체 페이지 로드 시간
  - 리소스 로딩 시간
  - JavaScript 실행 시간

### 2. 네트워크 분석

- **API 요청/응답 확인**
  - REST API 호출 모니터링
  - 응답 시간 측정
  - 에러 상태 코드 확인

- **리소스 로딩 최적화**
  - 이미지, CSS, JavaScript 로딩 시간
  - 캐싱 상태 확인
  - 병목 지점 식별

### 3. 콘솔 및 에러 디버깅

- **JavaScript 에러 확인**
  - 런타임 에러 캡처
  - 콘솔 경고 메시지
  - 네트워크 에러

- **로그 분석**
  - console.log 출력 확인
  - 디버그 메시지 추적

### 4. DOM 및 스타일 검사

- **요소 검사**
  - HTML 구조 분석
  - CSS 스타일 적용 상태
  - 레이아웃 문제 확인

- **접근성 검사**
  - ARIA 속성 확인
  - 시맨틱 HTML 검증

## 사용 예시

### 프론트엔드 개발 모드 테스트

```
1. 로컬 개발 서버 시작
2. http://localhost:5173 페이지의 성능을 분석해줘
3. 콘솔 에러가 있는지 확인해줘
4. 네트워크 요청 중 느린 API가 있는지 확인해줘
```

### 프로덕션 빌드 검증

```
1. 프로덕션 빌드 실행
2. http://localhost:4173 페이지의 LCP를 측정해줘
3. JavaScript 번들 크기가 적절한지 확인해줘
4. 이미지 최적화 상태를 확인해줘
```

### API 통합 테스트

```
1. 관리자 페이지(localhost:5173)로 이동해줘
2. 로그인 기능을 테스트하고 네트워크 요청을 확인해줘
3. 게시판 목록 API 응답 시간을 측정해줘
4. 에러 처리가 제대로 되는지 확인해줘
```

### 성능 최적화

```
1. 메인 페이지의 Core Web Vitals를 측정해줘
2. 가장 느린 리소스 5개를 찾아줘
3. JavaScript 실행 시간이 긴 함수를 찾아줘
4. 렌더링 블로킹 리소스를 확인해줘
```

## 주의사항

### 보안

- ⚠️ 원격 디버깅 모드에서는 민감한 정보 입력 금지
- ⚠️ 로컬 환경에서만 사용 (프로덕션 서버에 적용 금지)
- ⚠️ 테스트 계정만 사용

### 성능

- 디버깅 모드는 일반 브라우저보다 느릴 수 있음
- 정확한 성능 측정을 위해 시크릿 모드 사용 권장
- 브라우저 확장 프로그램 영향 최소화

### 포트 설정

- 기본 디버깅 포트: 9222
- 충돌 시 다른 포트 사용 가능
- 방화벽 설정 확인 필요

## 테스트 체크리스트

### 프론트엔드 관리자 (frontend-admin)

- [ ] 로그인 페이지 로드 성능 (LCP < 2.5s)
- [ ] 대시보드 API 응답 시간 (< 500ms)
- [ ] 게시판 목록 렌더링 성능
- [ ] 에디터 로딩 시간
- [ ] 이미지 업로드 기능
- [ ] 콘솔 에러 0개

### 프론트엔드 퍼블릭 (frontend-public)

- [ ] 메인 페이지 LCP (< 2.5s)
- [ ] 공지사항 목록 로딩
- [ ] 상세 페이지 전환 속도
- [ ] 모바일 반응형 테스트
- [ ] SEO 메타 태그 확인
- [ ] 콘솔 에러 0개

## 연관 명령어

- `ct`: Chrome MCP server 테스트 (짧은 버전)
- `commit`: 테스트 완료 후 변경사항 커밋

## 문제 해결

### Chrome이 연결되지 않을 때

```bash
# 기존 Chrome 프로세스 종료
pkill -f "Google Chrome"

# 디버깅 모드로 재시작
./scripts/start-chrome-debug.sh
```

### 포트 충돌

```bash
# 9222 포트 사용 중인 프로세스 확인
lsof -i :9222

# 해당 프로세스 종료 후 재시작
```

### MCP 연결 실패

1. Cursor 완전 재시작
2. `.cursor/mcp.json` 설정 확인
3. npx 캐시 정리: `npx clear-npx-cache`
