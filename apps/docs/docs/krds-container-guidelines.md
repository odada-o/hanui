# KRDS Container(컨테이너) 가이드라인

> 출처: KRDS (한국형 웹 콘텐츠 접근성 지침) - 디자인 스타일 > 레이아웃
>
> HANUI Container 컴포넌트는 이 가이드라인을 준수합니다.

## 컨테이너란?

컨테이너(Container)는 페이지의 콘텐츠를 제한된 너비 내에 정렬하고, 반응형 레이아웃을 구성하는 기본 레이아웃 컴포넌트입니다. 화면 크기와 상관없이 일관된 여백과 정렬을 제공하여 사용자 경험을 향상시킵니다.

## KRDS 레이아웃 시스템

KRDS는 그리드 시스템을 기반으로 한 레이아웃 가이드라인을 제공합니다.

### 그리드 요소

1. **Screen Margin (스크린 마진)**: 화면 양쪽 가장자리의 여백
2. **Column (칼럼)**: UI 요소를 수직으로 정렬하는 분할 영역
3. **Gutter (거터)**: 칼럼 사이의 간격

### 브레이크포인트

| 브레이크포인트 | 너비 범위 | 칼럼 수 | 거터    | 스크린 마진 |
| -------------- | --------- | ------- | ------- | ----------- |
| xsmall         | ~359px    | 4-6     | 16-16px | 16px        |
| small          | 360px~    | 4-6     | 16-16px | 16px        |
| medium         | 768px~    | 8-12    | 16-24px | 24px        |
| large          | 1024px~   | 12-16   | 16-24px | 24px        |
| xlarge         | 1280px~   | 12-16   | 16-24px | 24px        |
| xxlarge        | 1440px~   | 12-16   | 16-24px | 24px        |

## 용례

### ✅ 사용하기 적합한 경우

- 페이지의 메인 콘텐츠를 감싸는 경우
- 섹션별 콘텐츠를 일관되게 정렬해야 하는 경우
- 넓은 화면에서 콘텐츠가 너무 퍼지지 않도록 제한하는 경우
- 그리드 시스템의 래퍼로 사용하는 경우

### ❌ 사용하기 적합하지 않은 경우

- 전체 화면을 사용해야 하는 히어로 이미지나 배경 → `maxWidth="full"` 또는 `disableGutters` 사용
- Container 안에 Container를 중첩하는 경우 → 하나의 Container만 사용
- 이미 크기가 제한된 컴포넌트 내부 (Card, Modal 등)

## 사용성 가이드라인

### 1. 일관된 최대 너비 사용

사이트 전체에서 일관된 최대 너비를 사용하여 통일감을 제공합니다. 일반적으로 `lg` (1024px)가 권장됩니다.

### 2. 적절한 스크린 마진 유지

모바일에서는 최소 16px, 데스크톱에서는 24px 이상의 여백을 유지하여 콘텐츠가 화면 끝에 붙지 않도록 합니다.

### 3. 시맨틱 HTML 사용

`as` prop을 사용하여 적절한 시맨틱 요소를 선택합니다:

- `main`: 페이지의 주요 콘텐츠
- `section`: 섹션 그룹
- `article`: 독립적인 콘텐츠

### 4. 중첩 피하기

Container는 페이지당 한 번 또는 각 섹션당 한 번만 사용합니다. 중첩하면 불필요한 제약이 생깁니다.

### 5. 반응형 고려

모든 화면 크기에서 적절한 여백과 정렬을 유지할 수 있도록 KRDS 브레이크포인트를 따릅니다.
